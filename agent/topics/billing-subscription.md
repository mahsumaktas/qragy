# Billing & Subscription

This topic covers billing-related inquiries including plan changes, invoice questions, payment method updates, and license renewals. Many billing tasks can be resolved by guiding the user through the dashboard, but complex cases or disputes may require escalation.

## Type 1: Plan Upgrade or Downgrade

The user wants to change their current subscription plan, either upgrading to a higher tier or downgrading to a lower one.

### Flow

1. Direct the user to Settings > Billing in the dashboard.
2. Ask the user to click on "Change Plan" or "View Plans" to see available options.
3. Explain that upgrading takes effect immediately and the user will be charged a prorated amount for the remainder of the billing cycle.
4. Explain that downgrading takes effect at the end of the current billing cycle to avoid service interruption.
5. Ask the user to select the desired plan and confirm the change.
6. If the user has questions about feature differences between plans, provide a general overview and suggest checking the pricing page for details.

## Type 2: Invoice Questions

The user has a question about a past or upcoming invoice, such as unexpected charges, missing invoices, or invoice details.

### Flow

1. Direct the user to Settings > Billing > Invoice History.
2. Ask the user to locate the invoice in question and click on it to view details.
3. If the user sees an unexpected charge, explain common causes: mid-cycle upgrades, additional users added, or overages on usage-based features.
4. Suggest the user download the invoice as PDF for their records.
5. If the user disputes a charge or needs a correction, collect details and escalate to live support.

## Type 3: Payment Method Issues

The user needs to update their payment method, or a payment has failed.

### Flow

1. Direct the user to Settings > Billing > Payment Methods.
2. Ask the user to click "Update Payment Method" and enter the new card or payment details.
3. If a payment has failed, explain that the system will retry the payment automatically within a few days.
4. Suggest the user verify their card has not expired and has sufficient funds.
5. If the payment continues to fail after updating the method, collect details and escalate to live support.

## Type 4: License Renewal

The user's license is expiring or has expired and they need to renew it.

### Flow

1. Direct the user to Settings > Billing > Subscription.
2. Ask the user to check the renewal date and status of their current license.
3. If auto-renewal is enabled, inform the user that the license will renew automatically using the payment method on file.
4. If auto-renewal is disabled, ask the user to click "Renew" and follow the checkout process.
5. If the license has already expired and the user cannot access the platform, escalate to live support for manual reactivation.

## Required Information

- Organization name (if escalation is needed)
- Invoice number or billing period in question (for invoice disputes)

## Escalation

The bot can resolve most billing inquiries by guiding the user through the dashboard. Escalation to live support is needed for invoice disputes, refund requests, failed payments that persist after updating the payment method, and expired licenses that block access.

## Bot should

- Guide the user step by step through the Billing section of the dashboard.
- Explain prorated charges and billing cycle timing clearly.
- Help the user find and download invoices.
- Suggest updating the payment method when a payment fails.
- Escalate when the user disputes a charge or requests a refund.

## Bot should NOT

- Process payments, issue refunds, or apply credits directly.
- Share specific pricing amounts unless they are publicly available.
- Guarantee that a charge will be reversed or refunded.
- Access or display the user's payment card details.
- Make assumptions about the user's current plan without confirming first.
