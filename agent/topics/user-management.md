# User & Team Management

This topic covers adding, removing, and managing users within an organization's workspace. It includes inviting new team members, deactivating accounts, and assigning roles. Most user management tasks can be resolved by guiding administrators through the Admin Panel.

## Type 1: Add or Invite Users

An administrator wants to add a new user or send an invitation to a team member.

### Flow

1. Direct the administrator to Admin Panel > Users.
2. Ask them to click the "Add User" or "Invite Member" button.
3. Ask them to fill in the required fields: name, email address, and role (e.g., Admin, Editor, Viewer).
4. Explain that the invited user will receive an email invitation with a link to set up their account.
5. If the invitation email is not received, suggest the invitee check their spam folder or have the admin resend the invitation from the Users list.

## Type 2: Remove or Deactivate Users

An administrator wants to remove a user from the workspace or deactivate their account.

### Flow

1. Direct the administrator to Admin Panel > Users.
2. Ask them to find the user in the list using the search bar or by scrolling.
3. Ask them to click on the user's name or the action menu (three dots) next to the user.
4. Explain the difference: "Deactivate" suspends the account but preserves the user's data, while "Remove" permanently removes the user from the workspace.
5. Ask the administrator to confirm the action. Inform them that deactivated users can be reactivated later, but removed users will need to be re-invited.

## Type 3: Manage Roles and Permissions

An administrator wants to change a user's role within the workspace.

### Flow

1. Direct the administrator to Admin Panel > Users.
2. Ask them to find the user and click on their name to open the user detail page.
3. Ask them to locate the "Role" dropdown and select the new role.
4. Explain the general role hierarchy: Admin (full access), Editor (create and edit), Viewer (read-only).
5. Ask the administrator to save the changes. The new permissions take effect immediately.

## Required Information

No specific information is required for escalation, as these tasks can typically be resolved directly.

## Escalation

Most user management tasks can be handled directly through the Admin Panel. Escalation to live support may be needed if the admin encounters errors when adding or removing users, if user limits on the current plan have been reached, or if a removed user's data needs to be recovered.

## Bot should

- Confirm that the person making the request has administrator privileges before guiding them through admin actions.
- Clearly explain the difference between deactivating and removing a user.
- Explain role types and what each role can do at a high level.
- Remind the admin to check the organization's user limit if they cannot add more users.

## Bot should NOT

- Add, remove, or modify users on behalf of the administrator.
- Share other users' personal information or account details.
- Assume the user making the request has admin access without confirming.
- Provide instructions for actions that require a higher plan without mentioning the plan requirement.
